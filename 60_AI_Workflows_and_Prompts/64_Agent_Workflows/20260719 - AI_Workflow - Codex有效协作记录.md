# Codex 有效协作记录

> [!abstract]
> 本页沉淀已经确认有效的 Codex 协作模式，用于快速建立共同上下文、限定输出边界并将结果写入可维护的知识结构。

## 已确认有效的协作方式

### 知识库初始化

- 先说明目标是“让知识库理解我掌握的能力与方法”，而不是复制外部项目文件。
- 以“能力、方法、项目语境、当前边界、项目源路径”的结构建立领域页。
- 原始工程路径只放在笔记末尾作为证据与跳转入口；文件、数据集、模型和构建产物继续留在工程目录。
- 采用“全局能力地图 + 领域能力页”的双层结构：总图用于定位，领域页用于维护细节。

### 任务输入

- 直接给出目标、范围、不要做的事与期望落点。
- 提供项目文件夹、文献或截图作为事实来源；Codex 先匹配实际路径，再决定提炼方式。
- 对高风险内容明确要求“只做索引/待确认”，对低风险内容可要求直接落盘。

### 输出与维护

- 对科研内容分别记录方法、假设、实验/推导状态和可追溯来源，避免把方向描述成已验证结论。
- 使用 Obsidian 双链连接总览、领域索引和专题笔记。
- 只有稳定、可复用、经确认有效的协作模式才迁入本页。

## 可复用输入骨架

```text
目标：
范围：
希望知识库记住的能力/方法：
原始项目或资料路径：
不要做的事：
期望输出位置与维护方式：
```
